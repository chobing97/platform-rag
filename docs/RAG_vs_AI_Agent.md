# RAG vs AI Agent — 관계 정리

**작성일:** 2026.03.06
**맥락:** Knowledge Lake 아키텍처에서 RAG와 AI Agent의 관계

---

## 핵심 결론

**AI Agent ≠ RAG**이다. RAG는 AI Agent가 사용하는 **도구(패턴)** 중 하나이다.

---

## 1. 아키텍처에서의 관계

```
┌─────────────────────────────────────────────────┐
│              AI Agent (Layer 3)                   │

│   Agent = "목표를 달성하기 위해 스스로 도구를        │
│            선택하고 여러 단계를 수행하는 AI"         │
│                                                  │
│   ┌──────────────────────────────────────────┐   │
│   │  사용하는 도구들:                          │   │
│   │                                          │   │
│   │  ┌────────────┐  ┌────────────────────┐  │   │
│   │  │   RAG      │  │  기타 도구          │  │   │
│   │  │            │  │                    │  │   │
│   │  │  검색+생성  │  │  - 계산            │  │   │
│   │  │  (1회 사이클)│  │  - 메일 발송       │  │   │
│   │  │            │  │  - 파일 생성        │  │   │
│   │  └────────────┘  │  - 슬랙 메시지      │  │   │
│   │                  │  - 등등...          │  │   │
│   │                  └────────────────────┘  │   │
│   └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

---

## 2. RAG vs AI Agent 차이

| | RAG | AI Agent |
|---|---|---|
| **정의** | "검색해서 읽고 답변한다" (패턴) | "목표를 위해 스스로 판단하고 행동한다" (시스템) |
| **단계 수** | **1회 사이클:** 검색 → 답변 | **여러 사이클:** 검색 → 판단 → 추가 검색 → 비교 → 초안 작성 → ... |
| **판단** | 없음. 찾은 걸 요약할 뿐 | 있음. "이 정보가 부족하니 다른 검색을 해보자" |
| **행동** | 답변만 생성 | 답변 + 파일 저장, 메일 초안, 슬랙 알림 등 실행 |
| **비유** | 도서관에서 책 찾아서 읽어주는 사서 | 사서 + 변호사 + 비서를 겸하는 참모 |

## 3. 실제 작동 예시로 비교

### RAG만 있을 때 (단순 검색+답변)

```
CH: "ECACA §7.A에서 Clearing Broker의 의무는?"

1. 검색 (Retrieval)  → 관련 청크 5개 반환
2. 생성 (Generation) → "§7.A에 따르면 의무는 다음과 같습니다: ..."

끝. 1회 사이클.
```

### AI Agent일 때 (목표 지향적 다단계 수행)

```
CH: "IBKR이 우리를 Execution-Only로 분류한 건데, 반박 근거 만들어줘"

Agent 내부 동작:

  1단계 (RAG 1회차): ECACA 계약 조항 검색
  → "§7.A, §4A 찾았다. 그런데 Addendum도 확인해야겠다"
     ← Agent가 스스로 판단

  2단계 (RAG 2회차): Addendum 관련 조항 추가 검색
  → "Addendum 1 §3-L도 관련 있다"
     ← Agent가 정보 부족을 인식하고 추가 검색

  3단계 (RAG 3회차): 과거 IBKR 소통 이력 검색
  → "Stacy가 3/2에 보낸 메일에서 IBKR 측 입장 확인"
     ← Agent가 상대 입장도 파악해야 한다고 판단

  4단계 (추론): 세 번의 검색 결과를 종합
  → "Product Scope ≠ Clearing Relationship" 논거 구성
     ← 이건 검색이 아니라 Claude의 추론

  5단계 (생성): 반박 문서 초안 작성
  → 근거 3개를 인용한 구조화된 반박문 생성

  6단계 (행동): 결과물 저장
  → 분석 결과를 파일로 저장
```

**RAG는 위에서 1~3단계의 각 검색 사이클**이고, **AI Agent는 1~6단계 전체를 스스로 설계하고 실행하는 주체**이다.

## 4. 아키텍처 레이어에 매핑

```
┌──────────────────────────────────────────────────────┐
│  Layer 1: 수집 파이프라인                               │
│  (Notion, Gmail, 파일 → 파싱 → 임베딩 → 저장)          │
│  → RAG의 "데이터 준비" 단계                             │
└──────────────────────┬───────────────────────────────┘
                       │
┌──────────────────────┴───────────────────────────────┐
│  Layer 2: 검색 엔진                                    │
│  (BGE-M3 + Qdrant + Reranker)                        │
│  → RAG의 "R" (Retrieval)                              │
└──────────────────────┬───────────────────────────────┘
                       │
┌──────────────────────┴───────────────────────────────┐
│  Layer 3: AI Agent                                    │
│  ┌────────────────────────────────────────────────┐   │
│  │  Claude (LLM)                                  │   │
│  │                                                │   │
│  │  ┌──────────┐                                  │   │
│  │  │ RAG 호출 │ ← 필요할 때마다 Layer 2를 호출     │   │
│  │  │ (MCP)    │    (1회일 수도, 5회일 수도 있음)    │   │
│  │  └──────────┘                                  │   │
│  │  ┌──────────┐                                  │   │
│  │  │ 추론/판단 │ ← 검색 결과를 종합하여 분석        │   │
│  │  └──────────┘                                  │   │
│  │  ┌──────────┐                                  │   │
│  │  │ 행동/생성 │ ← 초안 작성, 파일 저장 등 실행     │   │
│  │  └──────────┘                                  │   │
│  └────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────┘
```

## 5. 정리

| 범위 | 구성 | 역할 |
|---|---|---|
| **Layer 1 + Layer 2** | 수집 파이프라인 + 검색 엔진 | RAG의 기반 인프라 (데이터 준비 + 검색) |
| **Layer 2 + Claude 1회 호출** | 검색 + 답변 생성 | RAG (검색 → 답변, 단순 질의응답) |
| **Layer 3 전체** | Claude + MCP + 다단계 추론 + 행동 | AI Agent (RAG를 도구로 활용하며 다단계 판단·실행) |

RAG를 먼저 정확하게 구축해야 Agent가 신뢰할 수 있는 판단을 내릴 수 있으므로, 구축 순서는 **Layer 1 → Layer 2 → Layer 3** 순서가 맞다.
